# 代码走查

## 概述

通过代码走查，找出代码中做得好的部分，找出做得不到位的部分

## 过程

1. 遍历所有的java，确保对代码之间联系有清晰认识
2. 重点走查可被复用/可被抽象的重复代码
3. 重点走查线程问题
   1. 线程创建必须命名
   2. 线程组必须存在关闭入口，且被调用
   3. 长时间运行的线程，必须考虑到因异常导致线程异常退出的情况
   4. 线程中使用的类，不得轻易修改成员变量，防止内存覆盖导致逻辑错误
   5. 线程中使用的ThreadLocal，需要考虑到内存泄漏问题
4. 重点走查事务问题
   1. dao只能注入到service，不得在其他类型中使用
   2. 纯内存操作，不应该开启事务
5. 对设计进行梳理，对使用的设计模式进行整理，对使用合理的进行指出表扬，对使用过度或不合理的进行指出批评

## 输出

每次结果都以中文输出到 `doc/review/java-review-{yyyymmdd}-{seq%000}.md` 中

## 核心原则

谦逊，目的是治病救人，目的不是羞辱人
