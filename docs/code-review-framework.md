# 代码审查框架规范

## 设计原则

将代码审查拆分为：
1. **通用审查框架**（本文件）：审查流程、原则、通用规则
2. **语言特定规则**（lang-spec 目录）：各语言的详细审查规则
3. **审查执行器**：根据项目语言加载对应规则集

## 文件结构

```text
${AI_SPEC_ROOT}/
├── skills/code-review/
│   └── SKILL.md              # 本文件 - 通用审查框架
├── lang-spec/                # 语言特定规则
│   ├── spec.java.md         # Java 审查规则集
│   ├── spec.python.md       # Python 审查规则集
│   ├── spec.cpp.md          # C++ 审查规则集
│   ├── spec.ansi_c.md       # ANSI C 审查规则集
│   ├── spec.rust.md         # Rust 审查规则集
│   ├── spec.js.md           # JavaScript/TypeScript 审查规则集
│   └── spec.shell.md        # Shell 脚本审查规则集
└── docs/                    # 审查结果输出
    └── review/
```

## 审查框架

### 1. 审查流程

1. **项目分析**
   ```bash
   # 识别项目语言
   Read ${AI_SPEC_ROOT}/lang-spec/spec.{detected_lang}.md
   ```

2. **加载语言规则**
   ```bash
   # 根据语言加载对应规则集
   Load language rules from: ${AI_SPEC_ROOT}/lang-spec/spec.{lang}.md
   ```

3. **执行审查**
   - 使用加载的规则进行代码审查
   - 应用通用审查框架

4. **生成报告**
   - 格式：`docs/review/code-review-{yyyymmdd}-{seq%000}.md`

### 2. 通用审查要点（所有语言适用）

#### 安全审查
- **敏感信息泄露**：密码、密钥、API Key硬编码
- **敏感数据内存残留**：内存中的敏感数据未清除
- **SQL注入**：字符串拼接SQL语句
- **XSS**：用户输入未转义

#### 性能审查
- **资源泄漏**：文件句柄、数据库连接未关闭
- **内存泄漏**：动态分配内存未释放
- **循环性能**：嵌套循环、N+1查询
- **对象创建**：循环中不必要的对象实例化

#### 代码质量
- **重复代码**：相同代码模式多处出现
- **方法长度**：方法 > 50行
- **条件分支**：圈复杂度过高
- **命名规范**：模糊命名、命名不一致

#### 并发安全
- **线程安全**：共享状态未同步
- **死锁风险**：锁顺序不当
- **资源竞争**：竞态条件
- **优雅退出**：线程可控制退出

### 3. 语言特定规则要求

每个语言规则文件必须包含：

1. **语言标识**：文件类型、扩展名
2. **语法特性**：该语言特有的语法检查点
3. **安全规则**：语言特定的安全问题
4. **性能规则**：语言特有的性能陷阱
5. **最佳实践**：语言编码规范
6. **审查示例**：问题代码和改进代码示例

### 4. 审查执行示例

```bash
# 示例：Java项目审查
# Step 1: 识别语言
lang="java"

# Step 2: 加载语言规则
Read ${AI_SPEC_ROOT}/lang-spec/spec.java.md

# Step 3: 执行通用审查
- 应用安全审查规则
- 应用性能审查规则  
- 应用代码质量规则

# Step 4: 执行语言特定审查
- 应用Java特有规则
- 检查Java语法特性
- 验证Java最佳实践

# Step 5: 生成报告
输出文件: docs/review/code-review-20250101-001-java.md
```

### 5. 扩展性设计

#### 新增语言支持
1. 在 `lang-spec/` 目录创建 `spec.{lang}.md`
2. 遵循语言规则模板格式
3. 更新语言支持列表

#### 规则更新
- 语言特定规则独立更新，不影响其他语言
- 通用框架更新，所有语言受益
- 向后兼容设计

#### 组合审查
支持多语言项目组合审查：
```bash
# 混合项目审查
langs=("java" "python" "javascript")
for lang in "${langs[@]}"; do
    Read ${AI_SPEC_ROOT}/lang-spec/spec.${lang}.md
    apply_language_rules $lang
done
```

### 7. 维护指南

#### 版本管理
- 语言规则文件独立版本控制
- 通用框架与语言规则解耦
- 兼容性测试：新规则不影响已有审查

#### 规则验证
```bash
# 验证规则文件格式
markdownlint lang-spec/spec.*.md

# 测试审查执行
模拟审查示例项目
```

#### 性能优化
- 按需加载语言规则
- 缓存常用规则集
- 增量审查支持

---

**核心优势**：
1. **上下文优化**：每次只加载需要的语言规则，减少LLM token负担
2. **维护简单**：语言规则独立修改，互不影响
3. **扩展灵活**：新增语言只需添加对应规则文件
4. **复用性强**：其他技能可复用语言规则集
5. **性能提升**：按需加载，减少内存占用
